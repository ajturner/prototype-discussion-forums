# CLAUDE.md

This file provides guidance for AI assistants working in this repository.

## Project Overview

This is a **Stencil Web Component prototype template** for building reusable UI components. It uses the [Stencil](https://stenciljs.com/) compiler (v2.x) to produce standard Web Components from TypeScript + JSX source. The Esri [Calcite design system](https://developers.arcgis.com/calcite-design-system/) is pre-configured as the UI component library.

The project ships as an NPM package (distributing `dist/` and `loader/`) and also builds a static `www/` site for local development/demo.

**Before using this template**, replace all occurrences of `REPLACE_PROJECT_NAME` in:
- `package.json` (`name`, `repository.url`)
- `stencil.config.ts` (`namespace`)
- `src/index.html` (script `src` attributes and `<title>`)

## Repository Structure

```
/
├── src/
│   ├── index.ts           # Package entry point — re-exports components and JSX types
│   ├── index.html         # Dev server HTML shell (dynamically mounts <prototype> component)
│   ├── components/        # Web Components (create with `npm run generate`)
│   │   └── <component>/
│   │       ├── <component>.tsx      # Component class + render
│   │       ├── <component>.scss     # Component styles (SASS)
│   │       ├── <component>.spec.ts  # Unit tests
│   │       └── <component>.e2e.ts   # End-to-end tests
│   └── components.d.ts    # Auto-generated by Stencil — do not edit
├── stencil.config.ts      # Build configuration
├── tsconfig.json          # TypeScript configuration
└── package.json
```

`src/components.d.ts` is machine-generated. Never edit it manually.

## Development Commands

```bash
npm start          # Dev server with hot reload (http://localhost:3333)
npm run build      # Production build + auto-generate README docs
npm test           # Run all spec and e2e tests once
npm run test.watch # Run tests in watch mode
npm run generate   # Interactive CLI to scaffold a new component
```

## Creating Components

Use the generator to scaffold new components:

```bash
npm run generate
```

This creates `src/components/<name>/` with `.tsx`, `.scss`, `.spec.ts`, and `.e2e.ts` files and registers the component automatically.

### Component Conventions

- **File naming:** `kebab-case` — both the directory name and the custom element tag must match (e.g., `discussion-thread` → `<discussion-thread>`).
- **Class naming:** `PascalCase` (e.g., `DiscussionThread`).
- **JSX factory:** Stencil uses `h` (not `React`). Import it from `@stencil/core`.
- **Decorators:** Use `@Component`, `@Prop`, `@State`, `@Event`, `@Listen`, `@Method`, `@Element` from `@stencil/core`.
- **Styles:** Each component has a co-located `.scss` file. Reference it via `styleUrl` in `@Component`.
- **Shadow DOM:** Enabled by default (`shadow: true`). Use `:host` selectors to style the host element.

Minimal component example:

```tsx
import { Component, Prop, h } from '@stencil/core';

@Component({
  tag: 'my-component',
  styleUrl: 'my-component.scss',
  shadow: true,
})
export class MyComponent {
  @Prop() name: string;

  render() {
    return <div>Hello, {this.name}</div>;
  }
}
```

## Styling

- SASS/SCSS is pre-configured via `@stencil/sass`.
- Use CSS custom properties (variables) for theming, especially when integrating Calcite tokens.
- Global styles go in `src/global/` (add `globalStyle` to `stencil.config.ts` if needed).

## State Management

`@stencil/store` is available as a dev dependency. For shared state across components:

```ts
import { createStore } from '@stencil/store';

const { state } = createStore({ count: 0 });
export default state;
```

## Calcite Components

Calcite components are loaded from CDN in `src/index.html`. To use them in component templates, reference them directly as JSX tags (e.g., `<calcite-button>`, `<calcite-panel>`). No import is needed since they self-register as custom elements.

The CSS is loaded from:
```html
<link rel="stylesheet" href="https://js.arcgis.com/calcite-components/1.0.7/calcite.css" />
```

Update this URL when upgrading `@esri/calcite-components`.

## Build Outputs

| Output | Directory | Purpose |
|--------|-----------|---------|
| `dist` | `dist/` | ESM + CJS for NPM consumers |
| `dist-custom-elements` | `dist/components/` | Tree-shakeable custom elements |
| `www` | `www/` | Static site for local dev/demo |
| `docs-readme` | `src/components/*/readme.md` | Auto-generated component docs |
| `docs-vscode` | `vscode-data.json` | VSCode IntelliSense data |

Data files (`src/data/*`) and HTML files are copied to `www/` during build.

## TypeScript

- Target: ES2017
- Strict unused-variable checks are enabled (`noUnusedLocals`, `noUnusedParameters`).
- JSX is handled by Stencil (`jsxFactory: "h"`). Do not set `"jsx": "react"`.
- Experimental decorators must remain enabled.

## Testing

- **Unit tests:** `*.spec.ts` using `@stencil/core/testing` (Jest-based).
- **E2E tests:** `*.e2e.ts` using Puppeteer via Stencil's test harness.
- Run a single spec: `npx stencil test --spec --e2e src/components/<name>/<name>.spec.ts`

## Key Constraints for AI Assistants

1. **Do not edit `src/components.d.ts`** — it is auto-generated by `stencil build`.
2. **Do not add `import React`** — Stencil uses `h`, not React.
3. **Replace `REPLACE_PROJECT_NAME`** before adding any meaningful code; leaving it causes broken build output paths.
4. **Keep output targets intact** in `stencil.config.ts` unless explicitly asked to change them.
5. **Co-locate component files** — each component lives in its own directory under `src/components/`.
6. **License:** Apache-2.0 — any new source files should carry the same license header if the project convention requires it.
