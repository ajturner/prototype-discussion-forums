## Project Purpose

This repository builds **prototype Chrome extensions and reusable Stencil Web Components** that integrate with the [ArcGIS Hub Discussions API](https://developers.arcgis.com/hub/api/discussions/). The goal is to embed discussion/forum UX into any web context (maps, dashboards, portals, third-party sites) and, over time, augment those discussions with LLM-powered moderation, summarization, and automated responses from live data queries.

### Planned Capability Areas

| Area | Description |
|------|-------------|
| **Discussion UI Components** | Reusable Web Components for threads, posts, reactions, channels |
| **Chrome Extension** | Injects discussion components into third-party or ArcGIS pages |
| **LLM Moderation** | Claude reviews posts for policy compliance before/after publish |
| **LLM Summarization** | Generates thread summaries and topic clusters from post data |
| **Automated Responses** | Claude queries live data (Feature Services, Hub content) to draft contextual replies |

## ArcGIS Hub Discussions API

Base URL: `https://hub.arcgis.com/api/discussions/v1`

### Key Resources

| Resource | Endpoint | Notes |
|----------|----------|-------|
| Channels | `GET /channels` | Scoped to a Hub site or item |
| Posts | `GET /channels/:id/posts` | Paginated, supports `sortBy`, `num`, `start` |
| Single Post | `GET /posts/:id` | Full post including reactions |
| Create Post | `POST /channels/:id/posts` | Requires auth token |
| Reactions | `POST /posts/:id/reactions` | Emoji reactions per user |
| Delete Post | `DELETE /posts/:id` | Author or moderator only |

### Authentication

All write operations require an ArcGIS token. Pass as:
- Header: `Authorization: Bearer <token>`
- or query param `token=<token>`

Obtain tokens via ArcGIS OAuth 2.0 (`arcgis-rest-auth` or `@arcgis/core/identity`). For Chrome extensions, store the token in `chrome.storage.local` — never in page cookies.

### Data Shape (Post)

```ts
interface HubPost {
  id: string;
  channelId: string;
  body: string;          // plain text or markdown
  creator: string;       // ArcGIS username
  createdAt: string;     // ISO 8601
  updatedAt: string;
  reactions: { [emoji: string]: number };
  status: 'active' | 'deleted' | 'flagged';
  parentId?: string;     // set for replies
}
```

## Stencil Web Component Conventions

### Naming

- Directory + tag: `kebab-case` (e.g. `discussion-thread` → `<discussion-thread>`)
- Class: `PascalCase` (e.g. `DiscussionThread`)
- JSX factory: `h` from `@stencil/core` — **never** `React`

### Decorators (import from `@stencil/core`)

| Decorator | Use |
|-----------|-----|
| `@Component` | Declares tag, styleUrl, shadow |
| `@Prop` | Public input — passed from parent or HTML attribute |
| `@State` | Internal reactive state |
| `@Event` | Emits custom DOM events upward |
| `@Listen` | Listens for DOM events |
| `@Method` | Exposes public async methods |
| `@Element` | Reference to the host element |

### Minimal Component

```tsx
import { Component, Prop, State, h } from '@stencil/core';

@Component({
  tag: 'discussion-post',
  styleUrl: 'discussion-post.scss',
  shadow: true,
})
export class DiscussionPost {
  @Prop() postId: string;
  @State() post: HubPost;

  async componentWillLoad() {
    this.post = await fetchPost(this.postId);
  }

  render() {
    return (
      <div class="post">
        <p>{this.post?.body}</p>
      </div>
    );
  }
}
```

### Shadow DOM

Shadow DOM is enabled by default. Style the host with `:host {}`. External styles cannot pierce the shadow boundary — use CSS custom properties for theming.

## Chrome Extension (Manifest V3)

### Structure

```
extension/
├── manifest.json          # V3 manifest — declarativeNetRequest, not webRequestBlocking
├── background/
│   └── service-worker.ts  # Persistent-free background (event-driven)
├── content/
│   └── inject.ts          # Injects <discussion-thread> into target pages
└── popup/
    └── popup.html         # Extension toolbar popup
```

### Key Patterns

- **Inject web components:** Load the Stencil component bundle as a content script resource, then `document.createElement('discussion-thread')` and append to the target DOM.
- **Messaging:** Use `chrome.runtime.sendMessage` / `chrome.runtime.onMessage` for content ↔ background communication.
- **Auth token storage:** `chrome.storage.local.set({ token })` from the popup after OAuth; content scripts read via `chrome.storage.local.get`.
- **Permissions:** Declare `storage`, `activeTab`, and any host permissions (`https://hub.arcgis.com/*`) in `manifest.json`.

### Manifest V3 Gotchas

- Background is a **service worker** — no DOM, no persistent state. Use `chrome.storage` for any state that must survive.
- `declarativeNetRequest` replaces `webRequest` for request modification.
- `chrome.action` replaces `chrome.browserAction`.

## LLM Integration (Planned)

Use the Claude API (`claude-sonnet-4-6` or `claude-opus-4-6`) for discussion augmentation features. Pattern:

```ts
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic(); // reads ANTHROPIC_API_KEY

// Moderation check
async function moderatePost(body: string): Promise<{ approved: boolean; reason: string }> {
  const msg = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 256,
    messages: [
      { role: 'user', content: `Review this forum post for policy compliance:\n\n${body}` }
    ],
  });
  // parse structured response
}

// Thread summarization
async function summarizeThread(posts: HubPost[]): Promise<string> {
  const content = posts.map(p => `${p.creator}: ${p.body}`).join('\n');
  const msg = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 512,
    messages: [{ role: 'user', content: `Summarize this discussion:\n\n${content}` }],
  });
  return (msg.content[0] as any).text;
}
```

### Planned LLM Features

| Feature | Trigger | Data Sources |
|---------|---------|-------------|
| Post moderation | Before/after publish | Post body, channel context |
| Thread summary | On-demand or periodic | All posts in a channel |
| Automated response | New post in monitored channel | Hub Feature Services, item metadata |
| Topic clustering | Scheduled / admin UI | All channel posts |

## Key Constraints for AI Assistants

1. **Do not edit `src/components.d.ts`** — auto-generated.
2. **Do not add `import React`** — Stencil uses `h`.
3. **Replace `REPLACE_PROJECT_NAME`** before adding meaningful code.
4. **Keep output targets intact** in `stencil.config.ts` unless asked.
5. **Co-locate component files** under `src/components/<name>/`.
6. **Never store ArcGIS tokens in page-accessible storage** — use `chrome.storage.local` in extensions.
7. **LLM calls are server-side or extension background only** — never expose `ANTHROPIC_API_KEY` to page content scripts.
